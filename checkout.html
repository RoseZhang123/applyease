ApplyEase，您的申请季小帮手。

产品介绍

  • 一键填表 约 2 分钟填完一份申请表，把本地 profile 写入支持学校页面。
  • 本地保存 资料只保存在您的浏览器中，不上传云端，不发送给 AI 服务。
  • 提交前可控 ApplyEase 不会代您提交申请、付款或上传材料，提交前始终由您核对。

⚠️ ApplyEase 是辅助填表工具，不替您做最终决策。学校表单有时会有改动，提交申请前请您务必细心核对所有字段。

使用流程

  1. 输入 activation code。激活后进入 ApplyEase 插件。
  2. 填写申请总表。可以下载本地 XLSX 模板填写后上传，也可以直接在插件内填写。
  3. 保存 profile。档案保存在您自己的浏览器中，后续可在插件内修改。
  4. 进入学校申请表页面。登录学校系统，打开具体 application form 页面。
  5. 点击 Start Filling / 开始填写。插件会识别页面字段，并把本地 profile 写入表单。
  6. 查看 sidepanel 提示。如有 warning、缺失字段或需手动处理的项目，sidepanel 会提示您逐项核对。

常见问题

Billing / 购买与续费

如何续费？

当前为人工续费流程。套餐到期前或到期后，您可以在本页确认个人版或企业版套餐，付款后把付款截图、所选套餐、微信或邮箱发送给客服。确认到账后，我们会发放新的续费激活码；若您在当前套餐到期前兑换，会从原到期日继续顺延。若套餐已过期，会从续费兑换时间重新开始计时。

是否提供免费使用？

暂不提供免费使用。ApplyEase 会直接辅助填写真实学校申请表，涉及持续适配学校页面、维护字段映射、处理技术支持和保障付费用户服务质量；基于产品本身的性质，我们现阶段只提供付费激活码。

一个 activation code 可以用几次？

每个 activation code 只能使用一次，且不可转让。激活成功后，该 code 会立即失效，不能再次用于其他设备或其他用户。

一个 activation code 支持几台设备？

一个 activation code 只支持一台设备。正式收费版本会将激活码绑定到首次激活的设备；如需更换设备，需要联系客服处理。

Technical / 功能与技术

支持填写哪些内容？

ApplyEase 主要支持从本地学生档案填写学校申请表中的结构化字段，例如个人信息、联系方式、教育经历、考试成绩、推荐人、工作/实习经历、奖项、活动、发表/研究经历，以及部分学校特定问题。

哪些内容暂不支持？

暂不支持文件上传、视频录制、付款页面、最终提交页、未适配学校页面，以及需要人工判断、创作或改写申请内容的场景。提交前仍需您自行核对全部信息。

个人版 XLSX 上传有什么限制？

个人版计划只允许使用一次 XLSX 资料模板上传，用于创建初始档案。完成首次上传后，后续资料修改需要在插件内编辑，避免频繁更换档案造成授权滥用。企业版的上传限制仍在设计中，正式上线前会在页面和插件内明确说明。

Customer / 售后与支持

遇到技术问题怎么办？

请通过客服提供购买时留下的微信或邮箱、插件版本、Chrome 版本、学校名称、页面类型和问题描述。若属于 ApplyEase 在已声明支持学校页面上的技术问题，我们会协助排查；因排查和修复耽误的有效使用天数，会在确认后补回。

如果问题导致核心自动填写功能无法在已声明支持的学校申请页面使用，且我们无法在合理时间内修复或提供替代方案，可申请退款。

需要提供哪些诊断信息？

通常需要购买时留下的微信或邮箱、插件版本、Chrome 版本、学校名称、页面类型和错误描述。请不要直接发送完整学生档案、证件号、文书、推荐信或包含敏感信息的截图。

Enterprise / 企业版

企业版适合谁？

企业版适合机构顾问集中管理多个学生档案，并在支持学校申请页面为不同学生执行自动填写。当前设计上限为 30 位学生档案。

企业版可以多人共用一个 activation code 吗？

不可以。企业版 activation code 仍然只支持一台设备，不能转发、共享或重复激活。后续如需多人席位，会单独设计 seat / account 模型。

Policies

政策说明

以下内容在购买前生效，请确认后再付款。

退款政策

激活码一经发放，原则上不支持退款。若因 ApplyEase 技术问题导致核心功能无法在已声明支持的学校申请页面使用，且我们无法在合理时间内修复或提供替代方案，可申请退款。

以下情况不支持退款：用户填写信息错误、学校系统临时变更但仍可手动提交、未按说明留下可联系的微信或邮箱、用户自行卸载或停止使用、学校政策或申请资格变化。

服务条款

ApplyEase 是申请表辅助填写工具，不替用户作出申请决定，不保证录取结果，也不会自动完成最终提交、付款、文件上传或视频录制。用户必须在提交前自行核对所有信息。

激活码仅限购买者或购买机构在授权范围内使用，不得转售、公开分享或用于未授权账号。每个 activation code 只能使用一次，且只支持一台设备；激活成功后不可转让。企业版最多管理 30 位学生档案。

隐私政策

ApplyEase 扩展内的学生申请资料默认保存在用户本地浏览器的 Chrome storage 中，不上传到 ApplyEase 服务器，不发送给 AI 服务。扩展会在用户点击开始填表后读取当前支持学校页面的表单结构，并根据本地档案写入表单字段。

购买激活码时，我们可能处理所选套餐、付款截图、邮箱、微信或支付宝联系信息，用于确认付款、发放激活码、处理技术支持和必要的售后沟通。我们不会出售用户数据，也不会将申请资料用于广告。

默认情况下，ApplyEase 不会自动上传诊断日志。若您遇到技术问题，我们可能请您主动导出并发送诊断信息；诊断信息应限于插件版本、Chrome 版本、学校、页面类型、错误类型和失败字段标识等排查所需信息，不应包含学生申请资料正文、证件号、联系方式、文书或截图中的敏感内容。

Use of information received from Google APIs will adhere to the Chrome Web Store User Data Policy, including the Limited Use requirements.

付费用户的普通技术支持请求我们会尽量在 3 个工作日内回复；若 Chrome Web Store 或 Google 将某项支持或产品问题标记为 urgent，我们会优先处理。
